## Section 1: Architecture Review

**13 decisions, all resolved.**

### A1 [P0] forge_logs live-tail hang -- RESOLVED
`forge logs <module>` without `--lines` is a blocking live tail. CLI docs confirm `--lines`
flag exists (`forge logs <module> --lines 200`). Fix: always pass `--lines min(max(lines, 1), 500)`.

### A2 [P1] forge_uninstall will hang -- RESOLVED
`run_forge("uninstall", module)` missing `--yes`. Fix: `run_forge("uninstall", module, "--yes")`.

### A3 [P1] forge_install timeout -- RESOLVED
Decision: per-tool timeout. `forge_install` and `forge_update` use `timeout=300`.
All other tools keep `timeout=60`.

### A4 [P1] forge_config_* absent -- RESOLVED
Add to core.py: `forge_config_show()`, `forge_config_get(key: str)`,
`forge_config_set(key: str, value: str)`.

### A5 [P1] SERVER_INSTRUCTIONS undefined -- RESOLVED
Written below in this document.

### A6 [P1] Startup validation behavior -- RESOLVED
Decision: fail fast. Lifespan hook runs `forge --version` at startup. On failure,
`print(f"FATAL: forge CLI not found: {e}", file=sys.stderr)` followed by `sys.exit(1)`.
Container exits with nonzero code. Docker restarts it. Docker logs show the clear error.
`/health` is only reachable when forge is available.

### A7 [P2] forge_secrets_set missing --sync -- RESOLVED
`forge_secrets_set(key: str, value: str, sync: bool = False)`. Passes `--sync` when True.
SERVER_INSTRUCTIONS notes: "If FluxForge is installed, pass sync=True to forge_secrets_set
to replicate secrets to all mesh nodes."

### A8 [P2] Missing __init__.py -- RESOLVED
Add `src/forge_mcp/__init__.py` and `src/forge_mcp/tools/__init__.py`.

### A9 [P2] forge_install missing --node -- RESOLVED
`forge_install(module: str, node: str = "")`. Passes `--node {node}` when non-empty.
Node is a FluxForge mesh IP (e.g., `10.forge.2.1`). Invalid values rejected by forge CLI.

### CT1 [P2] CLI contract drift -- RESOLVED
Provisional annotation format: `# PROVISIONAL: behavior not yet verified against Go CLI`.
Apply to any docstring asserting unverified behavior (e.g., "module starts automatically
after installation"). Remove annotation when Go implementation confirms behavior.

### CT2 [P2] refresh_tools() framing -- RESOLVED
Docstring: "Call after the MCP server image is rebuilt with new tools. Not needed after a
normal server restart -- Claude Code reconnects automatically. Sends ToolListChangedNotification
so Claude Code picks up new or changed tools without a manual /mcp reconnect."

### CT3 [INFO] Tool naming conventions -- ACCEPTED
Three conventions are intentional (CEO plan): `forge_*` for core, `<module>_*` for modules,
no prefix for meta tools (get_instructions, refresh_tools). forge_status() serves as the
capability discovery layer that resolves any ambiguity.

### CT4 [P2] Firewall documentation -- RESOLVED
No startup check. Add the following comment block to docker-compose.yml:

```yaml
# SECURITY: port 8008 must be restricted to localhost before production use.
# Run one of the following after starting the container:
#   iptables: iptables -A INPUT -p tcp --dport 8008 ! -s 127.0.0.1 -j DROP
#   ufw:      ufw deny in on any to any port 8008 && ufw allow in on lo to any port 8008
# Docker network_mode: host means any open port is open on the host.
```

## Section 4: Performance Review

### P1 [P1] forge_logs context flooding -- RESOLVED
500-line cap (CQ5) is the primary defense. Max output: ~100KB. Add to docstring:
"Returns up to 500 lines. Use `since` parameter (e.g., '1h', '30m') to narrow output."

### P2 [INFO] subprocess.run blocking -- ACCEPTABLE
All tool functions are sync `def` (not `async def`). FastMCP automatically wraps sync tools
in `asyncio.get_event_loop().run_in_executor()` (thread pool). subprocess.run blocking in
a thread does not block the event loop. No `asyncio.to_thread` calls needed in tool code.

Tool return contract: every tool function calls `fmt(run_forge(...))` and returns a `str`.
The `fmt()` function is the only place where `dict -> str` conversion happens. Tool modules
must not do their own JSON serialization.

### P3 [INFO] forge_status() on every session -- ACCEPTABLE
Sub-second latency once forge is installed. No performance concern.

## SERVER_INSTRUCTIONS (write this to server.py)

```python
SERVER_INSTRUCTIONS = """\
Forge MCP Server: manage your self-hosted infrastructure suite from Claude.

Forge is a suite of composable modules installed via a single CLI. This server
wraps the forge CLI so you can deploy apps, manage secrets, monitor services,
configure notifications, and run security scans without leaving your AI assistant.

IMPORTANT: Call forge_status() at the start of every session. It tells you which
modules are installed and running. Do not call module-specific tools (smeltforge_*,
watchforge_*, etc.) until you know the module is installed -- uninstalled modules
return a clear error but it wastes a round-trip.

Modules and their tools:
- Core (forge_*): module management, secrets, config, logs
- SmeltForge (smeltforge_*): deploy apps, manage CI/CD webhooks, rollback
- WatchForge (watchforge_*): uptime monitors, incident tracking, status page
- SparkForge (sparkforge_*): notification channels (Gotify, email, webhook)
- FluxForge (fluxforge_*): WireGuard mesh networking across multiple VPS nodes
- HearthForge (hearthforge_*): SSH dev workspaces for developers
- PenForge (penforge_*): automated security scanning (Nuclei, Nmap, testssl, Trivy)

Typical workflow:
1. forge_status() -- see what's installed and running
2. forge_install('smeltforge') -- install a module if needed
3. smeltforge_add(...) -- register your first app
4. smeltforge_deploy(...) -- deploy it

If FluxForge is installed across multiple VPS nodes: pass sync=True to forge_secrets_set
to replicate secrets to all mesh nodes. Without sync=True, secrets only land on the local node.

PenForge scans are long-running. Use penforge_scan_start() which returns a scan_id
immediately, then penforge_scan_status(scan_id) to poll for results.

Call get_instructions() to re-read these instructions after context compaction.
Call refresh_tools() after the server image is rebuilt with new tools -- this tells
Claude Code to refresh its tool list without needing a manual /mcp reconnect.
"""
```
