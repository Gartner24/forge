## Context

The Forge MCP Server is a Python/FastMCP sidecar service that wraps the Go `forge`
CLI binary via subprocess. It runs as a Docker container with `network_mode: host`
on the same VPS as the Forge suite. AI agents (Claude, etc.) connect via Streamable
HTTPS and issue natural-language infrastructure commands.

The Forge suite itself is entirely Go (7 modules) + Rust (HearthForge SSH gateway).
None of the Go modules are built yet. The MCP server wraps the CLI as it grows.

---

## Architecture Decisions

### Approach B: Full suite from day one
All 7 modules exposed. Uninstalled modules return graceful "not installed" errors.
The AI calls `forge_status()` first on every session to learn what's installed.

### Tool naming convention
- Core tools: `forge_` prefix (`forge_status`, `forge_install`, `forge_secrets_get`)
- Module tools: module prefix only (`smeltforge_deploy`, `watchforge_add`, `penforge_scan_start`)

### forge binary access
Bind-mount from host: `/usr/local/bin/forge:/usr/local/bin/forge:ro`
Docker container uses `network_mode: host` for network, bind-mount for forge binary.

### HearthForge CLI: flag-first design
`--output json` implies non-interactive. All inputs must be flags.
Prompts are a human-only convenience. See TODOS.md TODO-001.

### PenForge long scans: start/poll pattern
`penforge_scan_start(target_id)` returns scan_id immediately (requires `--async` flag in CLI).
`penforge_scan_status(scan_id)` polls for result.
See TODOS.md TODO-002 for CLI doc update.

### Startup validation
Server verifies `forge --version` on startup. Fails fast with a clear error if binary
is missing or not executable. Forge version exposed at `/health` endpoint.

### Capability discovery
`forge_status()` returns installed/running state for all modules. SERVER_INSTRUCTIONS
explicitly tells the AI to call it first on every session.

## Security Notes

- MCP port (8008) must be firewalled to localhost only. `network_mode: host` means
  any port open on the container is open on the host.
- `forge_secrets_get` returns plaintext values. This is intentional -- the MCP server
  IS the admin interface. Document and restrict port access.
- `hearthforge_gateway_add_key` injects SSH public keys. Admin-only operation. All
  MCP tool calls with side effects should be logged.
- Docker Desktop (Mac/Windows) is incompatible with `network_mode: host`. VPS-only.

---

## Platform Constraint

`network_mode: host` works on Linux only. Docker Desktop (Mac/Windows) silently falls
back to bridge networking, breaking the forge CLI access path. This service is
VPS-only by design. Document prominently in README and docker-compose.yml.

## What's NOT in Scope

- Web UI for the MCP server
- Cross-module orchestration shortcuts (e.g., `deploy_full_pipeline()` wrapping all modules)
- Auth layer on the MCP server endpoint (firewall is the auth layer for VPS deployments)
- Auto-generated tools from CLI spec (deferred until CLI is stable and proven)
- Docker Desktop compatibility shim
