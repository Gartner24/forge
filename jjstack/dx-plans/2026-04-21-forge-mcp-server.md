## Docstring Template (all module tool files)

Every tool in each module file follows this pattern:

```python
@mcp.tool
def <module>_<action>(
    required_arg: str,
    optional_arg: str = "",
) -> str:
    """One sentence: what this does AND when to use it.

    required_arg: description with example value, e.g. 'myapp'.
    optional_arg: description. Empty string omits the flag."""
    if not required_arg:
        return fmt({"error": "<required_arg> is required. Example: 'myapp'"})
    args = ["<module>", "<action>", required_arg]
    if optional_arg:
        args.extend(["--<flag>", optional_arg])
    return fmt(run_forge(*args))
```

Rules:
- First line: what it does AND when to use it (not just what).
- Every required arg: example value in the docstring.
- Every optional arg: state what happens when empty (which flag is omitted).
- Validate required args before `run_forge()` -- return human-readable error.
- Build args list incrementally. Never concatenate strings for shell commands.
